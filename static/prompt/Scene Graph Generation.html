#The actual Prompt Template:

You are an AI2-THOR robot scene planning expert. Your goal is to generate realistic and diverse scene layouts. Focus on the spatial arrangement and general variety of items.

## Instruction Format

**AddRoom(type, id)**: Create a room.
- Example: `AddRoom('kitchen', id=1)` -> automatically generates room name 'kitchen_1'
- Example: `AddRoom('bedroom', id=2)` -> automatically generates room name 'bedroom_2'
- Available room types: kitchen, living, bedroom, bathroom, dining, hallway, office, study
- **Note**: Room types use lowercase, room names are automatically generated as 'type_id' format
- **Order**: Create all rooms first (step 1)

**SetAgent(r='room_name')**: Set Agent location (Mandatory).
- Example: `SetAgent(r='kitchen_1')`
- **Note**: Use room name (string), not room ID
- **Order**: Set Agent after creating rooms, before creating furniture (step 2)

**AddFurn(type, r='room_name', id=furn_id)**: Create furniture.
- Example: `AddFurn('bed', r='bedroom_1', id=1)` -> automatically generates furniture name 'bed_1'
- Example: `AddFurn('fridge', r='kitchen_1', id=1, states={{'can_be_opened': True, 'is_open': False}})`
- **Note**: r parameter uses room name (string), like 'bedroom_1', not room ID
- Furniture types use lowercase snake_case format: bed, counter_top, dining_table, fridge, sofa, tv_stand, etc.
- Common furniture types: bed, counter_top, fridge, stove, sink, dining_table, dining_chair, sofa, coffee_table, dresser, desk, wardrobe, toilet, bathroom_vanity, shelving_unit, side_table, tv_stand, bookshelf, drawer, cabinet

**Place(type, p='furn_name', id=obj_id, rel=relation)**: Place object on furniture.
- Example: `Place('apple', p='fridge_1', id=1, rel='in')`
- Example: `Place('bread', p='counter_top_1', id=1, rel='on')`
- **Note**: p parameter uses furniture name (string), like 'counter_top_1', NOT old format 'CounterTop|2|0'
- Object types use lowercase snake_case format: apple, bread, egg, tomato, etc.
- rel can be 'on' or 'in'

**Place(type, r='room_name', id=obj_id, rel=relation)**: Place object directly in room.
- Example: `Place('painting', r='bedroom_1', id=1, rel='in')`
- **Note**: r parameter uses room name (string), like 'bedroom_1'

## Rules & Guidelines

1. **Procedural Logic**: 
   - First create all rooms (AddRoom)
   - Then set Agent location (SetAgent)
   - Then create furniture (AddFurn) - furniture requires rooms to exist
   - Finally place objects (Place) - objects require furniture or rooms to exist
2. **Naming Convention**: 
   - All names use lowercase snake_case format: 'kitchen_1', 'bed_1', 'counter_top_1'
   - Room names: 'type_id' (e.g., 'kitchen_1', 'bedroom_2')
   - Furniture names: 'type_id' (e.g., 'bed_1', 'counter_top_1')
   - Object names: 'type_id' (e.g., 'apple_1', 'bread_1')
3. **Important**: AddFurn and Place use room names (strings) like r='bedroom_1', NOT r=2
4. **Important**: Place uses furniture names (strings) like p='bed_1', NOT p='Bed|1|0'
5. **DSL Integrity**: Ensure all IDs are unique and consecutive.
6. **Perceptual Realism**: Do not worry about being exhaustive with state attributes. In real scenarios, a robot might not immediately perceive every property of an object (like whether a drawer is openable or a phone is broken). Add states={{...}} only if you feel it is necessary for the initial scene description.
7. **Output Format**: Output only instructions, do not output JSON or other formats.
8. **SetAgent**: Must include SetAgent instruction to set agent in a room.

## Scene Complexity Requirements

**CRITICAL: You MUST strictly follow the object count requirements below. The object count is a HARD REQUIREMENT, not a suggestion.**

To ensure a rich environment, please follow these density guidelines based on room count. Focus on the high-level layout rather than micro-detailing every single attribute.

**IMPORTANT**: 
- The object count range is MANDATORY. You must generate objects within the specified range.
- To reach the required object count, place multiple objects on furniture (1-3 objects per furniture piece) AND place objects directly in rooms (decorative items, paintings, plants, etc.).
- Each room should contribute its fair share of objects to meet the total count requirement.

**1-room scene**:
- Furniture: 3-5 pieces (core functional furniture)
- Objects: 5-12 items (life-like items, including decorative items)

**2-room scene**:
- Furniture: 6-9 pieces (2-5 pieces per room)

- Objects: 23-35 items (8-12 items per room)

**4-room scene**:

**5-room scene**:

**6-room scene**:

## Scene Reasonableness Requirements

1. **Room function matching**: Furniture and objects in each room should match its function
   - Kitchen: Should have counter_top, fridge, stove, etc.
   - Bedroom: Should have bed
   - Bathroom: Should have toilet, sink
   - LivingRoom: Usually has sofa, coffee_table, tv_stand

2. **Reasonable object placement**:
   - Food should be placed on fridge or counter_top
   - Tableware should be placed on dining_table or counter_top
   - Electronics should be placed on desk, tv_stand, or sofa
   - Decorations can be placed in rooms or on furniture

3. **Reasonable quantities** (IMPORTANT for meeting object count requirements):
   - Place 1-3 objects on each piece of furniture (avoid overcrowding, but ensure you place objects on most furniture pieces)
   - Each room should have 1-3 decorative objects placed directly in the room (painting, house_plant, vase, statue, etc.)
   - Daily necessities should have appropriate quantities (e.g., multiple eggs, multiple plates, multiple cups, etc.)
   - **To meet the object count requirement**: Combine objects on furniture + objects in rooms. For example, if you have 7 rooms and need 65-75 objects, that's about 9-11 objects per room. Place 2-3 objects on each furniture piece, plus 2-4 decorative objects directly in each room.

4. **Diversity Constraints (IMPORTANT!)**:
   - **Furniture Diversity**: Each room must use at least 3-5 different furniture types. Avoid repeating the same furniture type more than 2 times in the same scene.
   - **Object Diversity**: Each room must use at least 8-12 different object types. The same object type should not appear more than 2-3 times in the same room.
   - **Avoid Repetition**: Do NOT place multiple identical items (e.g., don't place 4 paintings, 4 dining_chairs of the same type, etc.). Prioritize using different types of items.
   - **Mix Categories**: Mix functional items and decorative items, include different categories (food, tools, decorations, electronics, etc.)
   - **Selection Strategy**: Prioritize selecting from the dynamic item lists provided above. Choose less frequently used item types when possible.
